# .agents/

Files and guardrails that help LLM coding agents use this project's frameworks, APIs, libraries, and interfaces correctly.

These are reference materials, not instructions about *how* an agent should behave or coordinate — agent-behavior rules live in the agent's own loader file (e.g. `CLAUDE.md`).

## Contents

- **`phoenix.md`** — Phoenix / LiveView / Ecto / HEEx usage rules, generated by Phoenix's `usage_rules` mix task. Re-sync with `docker compose run --rm web mix usage_rules.sync .agents/phoenix.md` when bumping framework versions.

## How it gets loaded

Whatever agent is in use imports the relevant files from its own loader. For Claude Code, `CLAUDE.md` does `@.agents/phoenix.md`.
