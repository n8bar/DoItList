# m02.04 — Collaborators (Wide-Rail)
_Parent: [`m02-ux-overhaul.md`](m02-ux-overhaul.md)_

On very wide screens (ultrawide / 4K), introduce a left rail that shows the user's Initiatives list and a cross-Initiative **Collaborators** section — everyone the user shares any Initiative with, deduplicated. Below the breakpoint, layout reverts to today's two-pane (main + right-rail Details); the rail does not appear.

All items are **draft — pending owner refinement and approval**. The shape below is a starting proposal; expect items to change before approval.

## Items

1. [ ] **Triple-pane layout at the wide breakpoint.** Pick a Tailwind breakpoint (likely `2xl:` or a custom one) above which the layout becomes left-rail · main · right-rail. Below the breakpoint, layout is unchanged from Arc 2. No layout shift when crossing the threshold.
2. [ ] **Left rail — Initiatives list.** Top of the rail. Lists all of the current user's Initiatives, each with the grove icon, the Initiative name, and a small progress indicator. Current Initiative is visually highlighted. Click navigates.
3. [ ] **Left rail — Collaborators section.** Below the Initiatives list. Aggregates everyone the current user shares any Initiative with, deduplicated. Sort heuristic to be decided (most recent shared activity? most shared Initiatives? alphabetical?). Each row: avatar or color blob, name, count of shared Initiatives.
4. [ ] **Click a Collaborator.** Opens a detail panel (right-rail or popover — to be decided) showing the Initiatives shared with that person. Action set is intentionally narrow for v1; future iterations can add direct-message, role management, etc.
5. [ ] **No new schema; pure aggregation query.** Reads existing memberships/initiatives tables. Cache or precompute later only if measured to be slow.
6. [ ] **Doesn't conflict with per-Initiative Members panel.** The right-rail Members panel (today's Initiative-scoped membership UI) stays as the canonical "who's on THIS Initiative" surface. The left-rail Collaborators section is a cross-Initiative roll-up — different question, different answer.
7. [ ] **Polished light + dark.** Both themes pass visual review at the new breakpoint.
8. [ ] **No layout shift entering or leaving the breakpoint.** Resizing the window across the threshold should be smooth — no layout jank, no scroll-position loss.

## Open Questions

- **Breakpoint.** Tailwind's default `2xl:` is 1536 px. Most ultrawides start around 2560 px wide. Pick the right boundary, or add a custom breakpoint.
- **Collaborator click behavior.** Right-rail panel? Popover? Navigate to a dedicated profile page? v1 scope intentionally minimal.
- **Initiative roles in the rail.** Do owner / editor / viewer badges from Arc 2 item 26 also appear next to each Initiative entry in the rail? Probably yes for consistency.
- **Empty state.** Solo user with no shared Initiatives — does the Collaborators section hide entirely, or render an empty-state hint?

## Implementation Notes

- **Aggregation query.** A single Ecto query joining `initiatives` → `initiative_members` → `users` should cover it. Group by collaborator user_id; count distinct shared initiative_ids.
- **Re-render frequency.** The rail's data changes only on membership writes — subscribe via PubSub on the existing membership topic rather than polling.
- **Selection state in the rail.** Current Initiative comes from the URL / LiveView assigns; no separate state.
