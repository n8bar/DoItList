# m02.04 — Account Details
_Parent: [`m02-ux-buildout.md`](m02-ux-buildout.md)_

A logged-in user needs one place to view and edit their own account — today the
only account surfaces are the register / log-in / log-out routes. This arc adds
an Account Details page.

## Preconditions
- Arc 3 (drag-and-drop) shipped; this arc is independent of it.

## Candidate scope (to confirm)
Existing `users` fields to expose: `email`, `name`, `theme` (password is
hashed). Likely items:

1. [ ] **Account page + route.** Authenticated `live "/account"` (or similar) reachable from the logged-in nav.
2. [ ] **Edit display name.**
3. [ ] **Change email** — with the usual confirm/verify flow if we want one.
4. [ ] **Change password.**
5. [ ] **Theme preference** — persist the light/dark/system choice to `users.theme` instead of (or alongside) localStorage.
6. [ ] **Delete account** — relationship to owned Initiatives TBD (ties into the Trash / ownership work in BACKLOG).
