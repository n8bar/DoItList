# m02.08 — Personal Workspace

_Parent: [`m02-ux-buildout.md`](m02-ux-buildout.md)_

Cross-Initiative surfaces — Assigned-to-Me worklist + Notifications feed — plus live
collaboration and per-user archiving.

**Depends on co-assignees ([`m02.05`](m02.05-wide-width-layout.md) item 12)** for the
direct-vs-co distinction; runs after Arc 5.

**Draft — pending operator approval.**

1. [ ] **Assigned to Me**

	1. [ ] **Page + route + nav item.** Authenticated `live "/assigned"` (or similar); new
	   **Assigned to Me** nav entry beside Initiatives (desktop nav + mobile hamburger). Title
	   "Assigned to Me", "My Initiatives" h1 style. Also fills the **ultrawide index right-pane
	   placeholder** ([`m02.05`](m02.05-wide-width-layout.md) item 6): at `3xl:` the index's
	   third pane (Details, on an open Initiative) is A2M's home. List built as a component for
	   both spots.
	2. [ ] **Cross-Initiative query.** Every task where user is **primary or co-assignee**,
	   scoped to Initiatives they're a **current member** of — left / removed / trashed drop
	   out via the current-member join (no special-casing). Completed **hidden by default**;
	   on-page toggle reveals.
	3. [ ] **Row display.** Distinguishes **direct (primary)** from **co-assignee**; shows
	   leaf/child count (mode-aware badge from `m02.04`-era work). **Flat — not expandable
	   here**; count conveys subtree size, no second tree UI.
	4. [ ] **Group by Initiative.** Persistent **Group by Initiative** checkbox (off by
	   default, flat); control on-page, state on `user_preferences` like the index sort
	   ([`m02.04`](m02.04-account-details.md) item 2.6) → follows the account.
	5. [ ] **Deep-link to task (the one new mechanism).** Row opens its Initiative landing,
	   task **selected, collapsed ancestors expanded, scrolled into view**. No
	   deep-link-to-task today (selection is client/DOM-owned) → new task param the show
	   LiveView reads on mount: select, expand ancestor chain, scroll. Honors
	   no-round-trip-before-input, no-layout-jank.

2. [ ] **Notifications**

	Per-user, cross-Initiative **unread feed** — distinct from the per-task Activity log
	(the full per-task record; this is the unread subset of what happened *to me*).

	1. [ ] New `notifications` table: `user_id`, `kind`, `data` (JSON), `read_at`, `inserted_at`.
	2. [ ] **Auto-generated** on: added / removed as Initiative member; **primary** assignee set/cleared on you; **co-assignee** added / removed (co counts as assigned). **Never notify the actor about their own action** — actor == recipient → skip. Hooks where we already broadcast `members_changed` / record activity (local additions, no new plumbing).
	3. [ ] **Flyout** in User menu (desktop dropdown) and hamburger (mobile): recent notifications newest-first, each linking its subject (the Initiative or — via this arc's deep-link, worklist 1 item 5 — the task); opening marks read; "mark all read" affordance.
	4. [ ] **Unread indicator:** **red dot** on the nav avatar while any unread exist; clears when read.
	5. [ ] **Real-time:** new notifications push live on a per-user PubSub topic; dot appears without reload.
	6. [ ] Tests: generation per event type, self-exclusion, unread count, mark-read, live push.

3. [ ] **Live collaboration**

	1. [ ] **Live chat for concurrent viewers.** Lightweight lower-left chat for everyone
	   currently viewing an Initiative — separate from per-task comment threads.
	   Presence-scoped to current viewers, fully ephemeral: no history, clears when last
	   viewer leaves. (Comments went live in [`m02.06`](m02.06-undo-redo.md) item 15.3; chat's
	   layout home set by Arc 7.)
	2. [ ] **Comment lifecycle — edit & delete.** Author can edit own comment (minimal popup
	   surfaces prior versions) or delete it, leaving a "comment deleted" tombstone (thread
	   shape + references survive). Needs version storage (`comment_versions` table or history
	   column) + a deleted marker (who / when); edits and deletes broadcast live. Non-authors
	   can't edit or delete.

4. [ ] **Archive (per-user).** A per-user flag moving any Initiative out of your active
   index / ultrawide rail into a restorable **Archived list**. Never touches anyone else's
   view; allowed on any Initiative regardless of completion or ownership. `archived_at` on
   your `initiative_members` row + the Archived list/filter. Distinct from Trash
   (owner-level, global, going away).
	1. [ ] **Archive offer on completion.** Hitting 100% *prompts* to archive — never
	   auto-archives.
	2. [ ] **Confirm only on unfinished work.** Archiving confirms first only if *you* have
	   incomplete assignments or co-assignments there; otherwise archives immediately.
	3. [ ] **Hidden (per-member) — subfeature.** A lighter "off my dashboard" hide, also
	   per-user (`hidden_at` on `initiative_members`); hidden Initiatives drop from the active
	   index, unhide through the same Archived list UI.

5. [ ] **Testing**

	1. [ ] **Tests.** Query correctness (primary + co; current-member scoping; completed
	   hidden/shown; left-Initiative tasks excluded); group-by persistence; deep-link selects +
	   reveals the right task; comment edit records a version and restores prior text, delete
	   leaves a tombstone; archive / hide drops an Initiative from that member's index only,
	   both restore via Archived list, archive confirms first with your own incomplete
	   assignments/cos there (live chat is presence/PubSub — [Human] below).
	2. [ ] **Usability testing** — [Human] eye-pass once built; mix tests ride each impl item.
		1. [ ] [Human] Tasks from several Initiatives appear; direct vs co reads at a glance; light+dark.
		2. [ ] [Human] Completed hidden by default; reveal toggle shows/hides.
		3. [ ] [Human] "Group by Initiative" groups, persists across reload, follows the account on another device/browser.
		4. [ ] [Human] Leaf/child counts read correctly; rows don't expand here.
		5. [ ] [Human] Row click opens its Initiative, task selected, ancestors expanded, scrolled into view — no jank.
		6. [ ] [Human] Task in an Initiative you've left drops out.
		7. [ ] [Human] Notifications: added/assigned by someone else raises red dot + flyout entry; self-actions never notify; opening flyout clears the dot; a live one raises it without reload.
		8. [ ] [Human] Live chat: two viewers of one Initiative see each other's messages live; fresh viewer sees no history; clears once last viewer leaves.
		9. [ ] [Human] Editing/deleting your own comment updates live for others — edit shows history popup, delete leaves tombstone; can't touch someone else's.
		10. [ ] [Human] Archive: archiving sets an Initiative into the Archived list (completing offers to archive); restore brings it back; confirm fires only with your own unfinished assignments/cos there.
		11. [ ] [Human] Hidden: hiding drops an Initiative from your dashboard only (others still see it); unhide via the Archived list.

## Cross-cutting requirements

- **Schema.** `notifications` table (worklist 2), comment-version storage (worklist 3
  item 2), `archived_at` + `hidden_at` on `initiative_members` (worklist 4), one boolean
  on `user_preferences` for the group-by toggle (worklist 1 item 4). Assigned-to-Me is a
  read over `tasks` + the co-assignee table + the membership join.
- **Light + dark visual review** of the new page and nav entry.
- **A11y.** List keyboard-navigable; deep-link open keyboard-reachable with visible focus;
  landed-on task takes focus / is announced.
